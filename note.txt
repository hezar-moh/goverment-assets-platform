# Demo User Management

Use the appropriate command based on your situation.

| Command | When to Use | What It Does |
|---------|-------------|--------------|
| `python manage.py setup_demo_data` | **First-time setup** or whenever you want to **reset the demo environment**. | Recreates Django demo users, demo assets, audit logs, **and** Keycloak users. |
| `python manage.py sync_keycloak_attribu` | When **Keycloak users already exist** but are missing the `role` and `ministry_schema` attributes (for example, after the Railway deployment issue). | Updates existing Keycloak users by adding the missing `role` and `ministry_schema` attributes without recreating the users. |
